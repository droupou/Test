Borland Compiler Installation of MySQL++

Installation

 

1.      Once you have downloaded the file named mysql++-1.7.1-win32-borland.zip. Extract it to wherever you would like. Once you extract it, the path is initially very long, I recommend renaming it to mysqlpp.  If you name it to mysql++, Borland has a hard time including files from this directory. This will be referred to as the ++root directory.

2.      You must have the Borland command line compiler (at least version 5.5) installed, if you do not have it installed you can obtain it from www.borland.com.  Borlands \bin directory must also be included in your PATH environment variable.

3.      In the ++root you will find the following structure:

 

++root\

\include\

\mysql – contains Borland specific interpretations of the interface to mysql

\rw – one file iterator.h (this must replace Borland’s iterator.h. (This will be explained later)

\sqlplus – will need to be included (might be a duplicate of the stuff in the \src directory.

\lib\

\deb – represents the debug libraries (used for debugging ++ apps)

\rel – supposedly stable libraries or release libs.  Not that stable.

\src\

\examples – some simple examples that work under Borland’s command line compiler.

\not_compiled – just as it states, examples that cant be compiled by Borland.

\sqlplus – these are the source of the .lib and .dll files found in the lib directory.

 

4.      Next you will need to build the examples for your machine.  To do this, the author of the library has provided a make file (++root\src\examples\examples.mak) for you which you will need to modify only slightly:

 

Find the line that matches the following:

 

BCB = D:\opt\CBuilder5

 

Change it to the directory of your Borland command line compiler directory.  It should look something like this:

 

BCB = c:\Borland\bcc55

 

You can also add a line right below the BCB line as follows (optional):

 

MYSQL = c:\mysqlpp

 

Which allows for the LIBFILES and INCLUDEPATH lines to look something like this:

 

LIBFILES = $(MYSQL)\lib\deb\libmysql.lib $(MYSQL)\lib\deb\mysqlplus.lib

INCLUDEPATH = $(BCB)\include;$(MYSQL)\include\mysql;$(MYSQL)\include\sqlplus

 

5.      Before actually running make, the iterator.h in the mysql++ library must replace the one in the Borland compiler directory.  To do this:

 

·        Find the file in ++root\include\rw.

·        Right-Click on the file iterator.h and select copy.

·        Find the directory [Borland Install Directory]\ BCC55\Include\Rw (i.e c:\Borland\ BCC55\Include\Rw)

·        Find the file named iterator.h and rename it to iterator.h.old

·        Right-Click anywhere in the folder, and choose paste.

 

The new iterator.h will now work to compile the mysql++ examples.

 

6.      Next, you will need to make the libraries in this directory. From a DOS prompt, type:

 

C:\mysqlpp\src\sqlplus> make -fmysqlplus.mak [ENTER]

 

This will run Borland’s make and attempt to compile, it will probably give you many warnings, ignore all warnings at this point.

 

Once compiled, you are ready to run any examples, create your own programs, etc.

 

GOOD LUCK!!!
